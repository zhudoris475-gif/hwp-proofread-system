## 3. 오류 원인

### 3-1. COM 자동화 실패 원인

- 원인 1: `0x800700c1`
  - 64비트 Python이 32비트 `HwpAutomation.dll`을 직접 로드하려 할 때 발생
  - 즉 `Dispatch()` 기본 경로는 비트수 불일치로 실패

- 원인 2: `서버 실행이 실패했습니다.`
  - `DispatchEx(..., clsctx=pythoncom.CLSCTX_LOCAL_SERVER)` 경로에서 `HwpApi.exe`가 DCOM 서버로 제시간 등록되지 못함
  - 시스템 이벤트 로그 기준 `DistributedCOM 10010` 확인
  - 의미: LocalServer 등록 경로도 현재 환경에서는 안정적으로 동작하지 않음

- 원인 3: 초기 실행 자원 누락
  - `olefile` 미설치
  - 작업 폴더 내 `rules_documentation.txt` 등 규칙 파일 부재
  - 기본 `HWP_DIR` 경로가 현재 PC 환경과 다름

### 3-2. 운영상 결과 차이 원인

- `.bak` 복원 여부에 따라 원본문 상태가 달라짐
- 복원 전 재실행 시 텍스트 자수와 교정 건수가 달라질 수 있음
- 따라서 실행 전 복원과 추출 자수 확인이 필수 절차임

## 4. 오류 해결 방침

### 4-1. 실행 환경 방침

- 한글 COM 자동화는 현재 PC에서 32비트 Python 기준으로 고정 운용
- 사용자는 `py -3`로 실행하더라도, 스크립트 내부에서 32비트 Python으로 자동 전환
- 32비트 Python 필수 모듈:
  - `pywin32`
  - `requests`
  - `olefile`

### 4-2. 파일 운용 방침

- 실제 적용 전 반드시 `.bak` 복원
- 복원 직후 텍스트 추출 자수 확인
- 규칙 파일은 작업 폴더에 두거나, 최신 기준 경로에서 동기화
- 로그 원문과 리포트를 함께 보존

### 4-3. 배포/점검 방침

- COM 생성 실패 시 먼저 비트수 경로를 점검
- LocalServer 회복보다 32비트 Python 경로를 우선 실무 경로로 사용
- 규칙 파일 누락 여부를 먼저 점검
- 결과가 달라질 경우 모델 성능보다 원본문 상태와 복원 여부를 먼저 확인
