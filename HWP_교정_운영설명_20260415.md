# HWP 통합규칙 교정 운영 설명서

## 1. 개요

HWP 파일의 띄어쓰기/붙여쓰기 통합규칙을 바이너리 수준에서 직접 교정하는 시스템입니다.
한컴오피스 COM 자동화 없이 olefile + zlib을 이용하여 HWP 파일의 압축된 BodyText 스트림을 수정합니다.

## 2. 파일 구성

| 파일 | 설명 |
|------|------|
| _integrated_proofread_binary.py | 메인 교정 스크립트 |
| rules_integrated.txt | 통합 교정 규칙 파일 |
| *.hwp.bak | 원본 백업 파일 (자동 생성) |
| _통합규칙_교정_보고서.txt | 교정 결과 보고서 (자동 생성) |

## 4. 규칙 파일 형식

rules_integrated.txt 파일 형식:

- 각 줄: 소스 -> 대상 형식
- 섹션 헤더(띄어쓰기, 붙여쓰기)는 자동 제외됨
- 동일 소스의 중복 규칙은 첫 번째만 적용됨

## 5. 처리 흐름

1. HWP 파일 읽기
2. OLE 스트림 분리 (BodyText/Section0 등)
3. zlib 압축 해제 (wbits=-15, 15, 31 순서 시도)
4. 실패 시 증분 압축 해제 (65536바이트 청크 단위)
5. HWP 레코드 파싱 (HWPTAG_PARA_TEXT = tagid 67)
6. 텍스트 교정 규칙 적용
7. 레코드 재직렬화
8. zlib 재압축 (raw deflate, wbits=-15) + 패딩
9. OLE 스트림 쓰기

## 6. 핵심 기술 사항

### 6.1 HWP 레코드 헤더 형식

4바이트 헤더:
- bit 0-9: tag_id (10비트)
- bit 10-19: level (10비트)
- bit 20-31: size (12비트, 0xFFF면 확장 헤더)

8바이트 확장 헤더:
- 4바이트: 위와 동일 (size=0xFFF)
- 4바이트: 실제 size (32비트)

### 6.2 압축 형식

- 표준: raw deflate (wbits=-15)
- zlib 헤더 포함: wbits=15
- 재압축 시 항상 raw deflate(wbits=-15)로 통일
- 원본 압축 크기에 맞추어 zero 패딩 추가

### 6.3 텍스트 인코딩

- HWP 텍스트 레코드(tagid=67)는 UTF-16LE 인코딩
- 각 문자는 2바이트, null 종료 문자 포함

## 7. 주의사항

### 7.1 규칙 충돌

규칙 파일에 서로 모순되는 규칙이 존재할 수 있습니다:
- 책 읽기 -> 책읽기 (띄어쓰기->붙여쓰기 섹션)
- 책읽기 -> 책 읽기 (붙여쓰기->띄어쓰기 섹션)

첫 번째로 로드된 규칙이 우선 적용됩니다.
충돌 규칙은 규칙 파일에서 수동으로 제거해야 합니다.

### 7.2 백업

스크립트 실행 시 자동으로 .bak 파일이 생성됩니다.
문제 발생 시 백업 파일로 복원:

```powershell
copy "원본.hwp.bak" "원본.hwp"
```

### 7.3 압축 크기 제한

수정된 데이터의 압축 크기가 원본보다 크면 쓰기가 실패합니다.
압축 레벨 1~9를 순차적으로 시도하며, 모두 실패하면 오류가 보고됩니다.

### 7.4 다중 세그먼트 압축

일부 HWP 파일은 여러 개의 독립적인 압축 세그먼트를 사용합니다.
증분 압축 해제로 첫 번째 세그먼트만 처리되므로 전체 텍스트가 아닐 수 있습니다.
이 경우 한컴오피스에서 직접 교정해야 합니다.
