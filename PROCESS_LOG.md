## 1단계: 초기 비교 스크립트 개발 (compare_hwp.py)

### 목표
HWP 파일의 텍스트를 추출하여 원본과 교정본 비교

### 접근 방식
1. `olefile` 라이브러리로 HWP OLE 스트림 열기
2. `BodyText/Section{N}` 스트림에서 zlib 압축 해제
3. UTF-16-LE 디코딩으로 텍스트 추출
4. difflib.SequenceMatcher로 텍스트 비교

### 문제점
- HWP 바이너리 노이즈가 텍스트에 포함됨
- 노이즈가 "삭제된 중국어"로 잘못 식별됨
- 예: `猴盔穴縔膴`, `厔在媼幐懤` 등은 실제 중국어가 아닌 바이너리 노이즈

## 2단계: 노이즈 필터링 개선 (compare_hwp_v2 ~ v3)

### 개선 내용
1. **U+XX00/U+XX04 패턴 탐지**: 한국어 음절 중 Unicode 하위 바이트가 0x00 또는 0x04인 문자를 노이즈로 식별
2. **빈도 기반 노이즈 탐지**: 출현 빈도가 비정상적으로 높은 중국어 문자를 노이즈로 분류
3. **의미 있는 내용 비율 분석**: 중국어 문자 구간에서 실제 의미 있는 문자의 비율이 낮으면 노이즈로 판단
4. **알려진 노이즈 구문 제거**: "문단띠로", "사각형입니다" 등 HWP 메타데이터 노이즈 제거

### 결과
- 초기: 253개 "실제 내용 변경" (대부분 노이즈)
- 중국어 노이즈 필터링 후: 삭제된 중국어 0개 달성

## 3단계: 한국어 띄어쓰기 변경 감지 (generate_recovery_list.py v1)

### 문제
- 띄어쓰기 변경(예: "사이 에" → "사이에")이 content change로 잘못 분류됨
- 단어 분할 손실: "가운데서"가 "가운데"+"서"로 분할되지 않아 "데서" 삭제/"한데" 추가로 오분류

### 접근 방식
1. 한국어 단어 추출: `[가-힣]+` 정규식
2. 유효 단어 집합 구축: 원본+교정본에서 2회 이상 등장한 한국어 단어
3. 단어 분할(resolve_words): 사전 기반 최장 일치 분할
4. 문자열 비교: 분할된 단어를 연결한 문자열이 같으면 띄어쓰기 변경, 다르면 내용 변경

### 결과
- 띄어쓰기 변경: 2개, 실제 내용 변경: 19개

## 4단계: kr_ratio 유사도 검사 도입 (generate_recovery_list.py v2)

### 문제
- HWP 노이즈로 인해 원본/교정본의 한국어 문자가 미세하게 다름
- 예: 【信】 "데서"→"한데" (실제로는 "한데서"의 분할 오류)
- 원본: 202자, 교정본: 205자 (3개 노이즈 문자 차이)
- kr_ratio = 0.9484

### 해결
1. 한국어 문자만 추출하여 SequenceMatcher로 유사도 계산
2. kr_ratio >= 0.94이면 띄어쓰기 변경으로 분류
3. HWP 노이즈로 인한 1~3개 문자 차이를 허용
